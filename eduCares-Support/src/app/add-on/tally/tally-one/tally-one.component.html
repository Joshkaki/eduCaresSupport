What is the advantage of Tally integration?

Most of the institutions are using Tally to maintain their financial system. Whereas in EduCares, user can make any type of finance transactions. Student can pay Fees (i.e. tuition fee, library fee, transport fee etc.); Admin/authorized employee can issue salary for other employees etc. So, it’s very absurd to enter manually all the finance transactions from EduCares to Tally.EduCares Tally is a Plugin. It will copy all the finance transactions from EduCares to Tally. Basically, it’s a bridge between EduCares and Tally. There are three type of synchronization mode in this plug in.

  1. Live Sync
  2. Manual Sync
  3. Bulk Export

If you set as Live Sync, as in the image below.

If Live sync is enabled, then for each transaction it will send http request to your Tally server and your Tally system will be updated automatically.

If you set as Manual Sync then you have to send the http request manually. You need to select start date and end date, as in Fig 2 and click on schedule; it will send a http request to your Tally server and your Tally system will be updated automatically.

And for Bulk export mode, you need to select start date and end date and click on Export; it will generate a xml file. Download the file from download section and import it to your Tally.

N.B. For live sync and manual sync, you need to configure your tally as a server. For that open Tally and go to Configuration, then Advanced Configuration and configure it as mentioned below.

  • Tally is acting as : Server/Both
  • Port : 9000
